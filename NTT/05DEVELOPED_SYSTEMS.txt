This section describes four different systems that we developed for the CHiME-3 challenge by exploiting the techniques described in the previous sections.
このセクションでは、前のセクションで説明した手法を利用してCHiME-3チャレンジのために開発した4つの異なるシステムについて説明します。

The systems include a one-pass SI system, a multi-pass SI system, a single-model SA system, and a multi-model SA system.システムには、ワンパスSIシステム、マルチパスSIシステム、シングルモデルSAシステム、マルチモデルSAシステムがあります。

The first three constrained systems are described in Section 5.1.
最初の3つの制約されたシステムはセクション5.1で説明されます。

The impact of speaker adaptation on NIN-CNN acoustic models is also described.
話者適応がNIN-CNN音響モデルに与える影響についても説明します。

The last system represents the NTT CHiME-3 system and is described in Section 5.2.
最後のシステムはNTT CHiME-3システムを表し、セクション5.2で説明します。

We focus on the performance on the real parts of the development and evaluation sets.
我々は、開発と評価のセットの実際の部分のパフォーマンスに焦点を当てます。

5.1.Constrained systems
制約されたシステム

Our one-pass SI system is based on an NIN-CNN acoustic model built with multi-microphone training, a word class-based RNN-LM, and a one-pass decoder with on-the-fly rescoring [12] as described in Section 3.
私たちのワンパスSIシステムは、マルチ・マイク・トレーニング、単語クラス・ベースのRNN-LM、およびセクション3で説明したオンザフライの再圧縮[12]を備えたワンパスデコーダとがあります。

This system does not perform speech enhancement.
このシステムは、音声強調を実行しません。

The development and evaluation WERs of this system are shown on the “1-pass SI” rows in Tables 7 and 8, respectively.
このシステムの開発および評価WERは、それぞれ表7および8の「1パスSI」行に示されています。

These results correspond to those of Section 3.3.
これらの結果は3.3節の結果と一致する。

The average WERs were 9.01% and 15.60% for the real parts of the development and evaluation sets, respectively.
平均WERは、開発および評価セットの実際の部分でそれぞれ9.01％および15.60％であった。

These numbers are better than the official baseline WERs (16.13% for dev and 33.43% for eval) by 44.1% and 53.3% relative, respectively.
これらの数値は、公式のベースラインWER（devの場合は16.13％、evalの場合は33.43％）よりもそれぞれ相対的に44.1％と53.3％高い数値です。


Our multi-pass SI system performs SI decoding using enhanced features obtained by performing dereverberation and beamforming.
我々のマルチパスSIシステムは、残響除去とビームフォーミングを実行することによって得られる強化された特徴を使用してSIデコードを実行する。

Note that decoding is performed with a one-pass approach while enhancement processing scans each utterance multiple times.
デコードはワンパス・アプローチで行われ、エンハンス処理は各発話を複数回スキャンすることに注意してください。

In Tables 7 and 8, the “multi-pass SI” rows show the WERs of this system.
表7および8において、「マルチパスSI」の行は、このシステムのWERを示す。

The system achieved WERs of 4.67% and 8.32% for the development and evaluation sets, respectively.
このシステムは、開発および評価セットでそれぞれ4.67％および8.32％のWERを達成した。

The relative gains from speech enhancement were 48.2% and 46.7% for the development and evaluation sets, respectively.
音声強調の相対的な利得は、開発および評価セットでそれぞれ48.2％および46.7％であった。


In our single-model SA system, unsupervised speaker adaptation is performed just once as shown in Fig.3.
我々の単一モデルSAシステムでは、教師なしの話者適応は図3に示すように1回だけ実行される。

The system adapts the SI NIN-CNN acoustic model to individual speakers by using adaptation supervision generated by the multi-pass SI system.
システムは、マルチパスSIシステムによって生成された適応監督を用いて、SI NIN-CNN音響モデルを個々の話者に適合させる。

In this SA system, the model adaptation step aims at compensating for both speaker differences and acoustic changes resulting from front-end enhancement processing.
このSAシステムでは、モデル適応ステップは、フロントエンドエンハンスメント処理に起因するスピーカの相違および音響変化の両方を補償することを目的とする。

Adaptation is performed by re-training the neural network with SGD on a modest number (i.e., 1800) of minibatches with a fixed learning rate of 0.02 [36].
適応は、0.02 [36]の固定学習率で、適度な数（すなわち、1800）のミニバッチでSGDを用いてニューラルネットワークを再トレーニングすることによって実行される。


In Tables 7 and 8, the “1-model SA” rows show the WERs of the single-model SA system.
表7および表8において、「1-モデルSA」の列は、単一モデルSAシステムのWERを示す。

The system achieved a 3.90% development error rate and a 6.58% evaluation error rate.
システムは、3.90％の開発エラー率と6.58％の評価エラー率を達成しました。

The gains from speaker adaptation were 16.5% and 21.0% relative for the development and evaluation sets, respectivley.
話者の適応による利得は、開発セットと評価セットでそれぞれ16.5％と21.0％でした。

This result means that NIN-CNN models can benefit significantly from speaker adaptation.
この結果は、NIN-CNNモデルが話者適応から大幅に利益を得ることを意味します。


5.2.Submitted system
提出されたシステム

The architecture of our system submitted to the CHiME-3 challenge centres around system combination with cross-adaptation.
CHiME-3チャレンジに提出されたシステムのアーキテクチャは、クロスアダプテーションによるシステムコンビネーションを中心としています。

Combining multiple complementary models has been proven to result in lower WERs by many evaluation systems [13, 37, 38].複数の補完モデルを組み合わせることは、多くの評価システムによってWERが低下することが証明されている[13,37,38]。

With the cross adaptation approach, outputs from one model are used to modify another model.
クロスアダプテーションアプローチでは、あるモデルの出力を使用して別のモデルを修正します。

Figure 4 shows the way in which our submitted system performs decoding and adaptation.
図4は、提出されたシステムがデコードとアダプテーションを実行する方法を示しています。

Three acoustic models from Table 3 are used: RBM-DNN10, CNN3, and NIN-CNN.
RBM-DNN10、CNN3、およびNIN-CNNの3つの音響モデルが使用されています。

The system processes test utterances as follows:
システムは、テスト発話を以下のように処理する。

SI1 SI decoding with the RBM-DNN10 model.
    SI RBM-DNN10モデルによるデコード。

SA1 Adaptation and decoding with the NIN-CNN model by using supervisions created at Step SI1.
    ステップSI1で作成された監督を用いて、NIN-CNNモデルによる適応および復号。
    
SA2 Adaptation and decoding with the CNN3 model by using supervisions created at Step SA1.
    ステップSA1で作成された監督を使用して、CNN3モデルによる適応および復号。
    

The WERs of the respective decoding steps are shown on the SI1, SA1, and SA2 columns in Table 9.
それぞれの復号ステップのWERは、表9のSI1、SA1、およびSA2の欄に示されている。

We can see that the performance gradually improved step by step while the WER for eval-real saturated in Step SA1.
ステップSA1で評価された実数のWERが飽和している間に、段階的に性能が徐々に向上することがわかる。


To confirm that cross adaptation is effective for DNN-HMM acoustic models, we compared the WERs of the cross-adapted NINCNN with a self-adapted version that used supervisions generated by an SI NIN-CNN model.
交差適応がDNN-HMM音響モデルに有効であることを確認するために、クロス適合NINCNNのWERを、SI NIN-CNNモデルによって生成された監督を使用する自己適合バージョンと比較した。

Table 10 shows the results.
結果を表10に示す。

Although the transcripts generated by the SI RBM-DNN10 model contained more errors than those generated by the SI NIN-CNN model, adapting the NIN-CNN model using the former transcripts resulted in lower WERs.
SI RBM-DNN10モデルによって生成された転写物は、SI NIN-CNNモデルによって生成された転写物よりも多くのエラーを含んだが、前転写物を用いてNIN-CNNモデルを適合させると、WERが低下した。


The SA2+ system in Table 9 represents the NTT CHiME-3 system, which was built at the final stage of our system development by tuning some hyper-parameters for Step SA2 (using the development set).
表9のSA2 +システムは、（開発セットを使用して）ステップSA2のいくつかのハイパーパラメータを調整することによって、システム開発の最終段階で構築されたNTT CHiME-3システムを表しています。

The hyper-parameters adjusted at this stage include the number of mini-batches used for adaptation, an adaptation learning rate, and a language model scale.
この段階で調整されるハイパーパラメータは、適応に使用されるミニバッチの数、適応学習率、および言語モデルのスケールを含む。
The optimal values for the repsective parameters were 2700, 0.01, and 12.
それぞれのパラメータの最適値は2700,0.01、および12であった。

In addition, SA2+ uses a shorter analysis window for beamforming than SA2 to improve complementarity with the SA1 model (64 msec for SI/SA1/SA2 and 25 msec for SA2+).
さらに、SA2 +は、SA1モデル（SI / SA1 / SA2の場合は64ミリ秒、SA2 +の場合は25ミリ秒）との相補性を向上させるために、SA2よりも短いビーム形成ウィンドウを使用します。

This system achieved a development error rate of 3.45% and an evaluation error rate of 5.83%.
このシステムは、3.45％の開発エラー率と5.83％の評価エラー率を達成しました。

This means that recognition errors were reduced by 11.5% and 11.4% for the development and evaluation sets, respectively, compared with the single-model SA system.
これは、単一モデルのSAシステムと比較して、開発および評価セットでそれぞれ11.5％および11.4％の認識エラーが削減されたことを意味します。

提出されたシステムのWER、すなわちSA2 +は、個々の条件について表11に示されている。
